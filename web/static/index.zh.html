language

在线测试

  1. 选择源位置（全球 / 区域 / 国家 / 城市）和目标端点
  2. 从所选位置的志愿者探针节点发出真实的 ICMP 或 HTTP 测量请求
  3. 结果实时流式返回 — 包括延迟、丢包率和完整的 HTTP 时间分解
  4. 所有测量均来自真实的 ISP 级别网络路径，而非模拟数据
leaderboard

排行榜

  1. 每个测试周期从各国最多 10 个城市同时采样所有云厂商 Region，确保跨厂商对比公平可信
  2. 按 (国家, 厂商, 区域) 分组聚合中位延迟和丢包率
  3. 按加权中位延迟排名 — 越低越好
  4. 排行榜覆盖 3 天、15 天和 30 天时间窗口，每天更新
  5. 按区域排行榜：选择地理区域时，收集该区域内所有国家的全部云厂商 Region，仅保留超过半数国家中出现的云厂商 Region（多数交集），平均延迟和中位延迟取各国平均值，丢包率取平均值，城市数和样本数取求和值，最终取前 20 名

常见问题

测量来自志愿者探针上真实的网络路径，覆盖多种 ISP。这提供了真实的终端用户视角，但测试结果可能因网络状况、探针负载和路由变化而有所波动。中位数和多日聚合有助于平滑异常值。

PingCloud 目前覆盖 7 大主流云厂商：AWS、Google Cloud (GCP)、Microsoft Azure、阿里云、腾讯云、华为云 和 Oracle Cloud。共测试超过 150 个云区域。

后端每天持续运行测试周期。排名文件在每个周期后重新生成，因此排行榜通常反映过去几小时的数据。您可以在排行榜上查看「更新时间」。

是的，在线测试完全免费，无需注册。

这意味着所选位置当前没有在线的志愿者探针。探针可用性取决于志愿者参与度，因地区而异。请尝试选择更大的范围（例如选择国家或区域，而非特定城市）。

当一个国家对同一云区域有多个测试城市时，每个城市的中位数按其样本数量加权。这防止了测试次数较少的城市扭曲结果 — 数据量越大的城市对最终排名的影响越大。
